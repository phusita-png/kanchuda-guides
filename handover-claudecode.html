02
สมัคร & เลือกแพ็กเกจ

สมัครบัญชี Claude และเปิดแพ็กเกจ Pro

ใช้อีเมลอะไรก็ได้ · แพ็กเกจ Pro $20/เดือน ใช้ Claude Code ได้เพียงพอสำหรับดูแลบอทกานต์ชุดา

เข้า claude.ai แล้วเลือกวิธีสมัคร

เปิด Chrome → เข้า claude.ai/login → เลือก “Continue with Google” (สมัครด้วย Gmail เร็วสุด แนะนำ) หรือ “Continue with email” กรอกอีเมลรับลิงก์ยืนยัน

💡Sign up = สมัครใหม่ · Log in = เข้าสู่ระบบ (ถ้ามีบัญชีแล้ว)

เปิดเมนูอัปเกรดแพ็กเกจ

เมื่อเข้าใช้งานแล้ว → คลิกที่ ชื่อผู้ใช้/อวตาร์ มุมซ้ายล่าง → เมนูจะเด้งขึ้น → คลิก “Upgrade plan”

เลือกแพ็กเกจ Pro

เลือกแท็บ Individual → เลือก Pro (เริ่มต้นแนะนำ Pro ก่อน) → เลือก Monthly (รายเดือน) → กด Try Claude

💳ตัดเป็นดอลลาร์ (~$17–20 ≈ 600–700 บาท/เดือน) · ยกเลิกเมื่อไหร่ก็ได้
✨
ทำไม Pro ก็พอ: ดูแลบอทเป็นการแก้เป็นครั้งคราว โควต้า Pro รับไหวสบาย — ถ้าอนาคตแก้บ่อยจนโควต้าหมดเร็ว ค่อยขยับเป็น Max ทีหลังได้ (feature เหมือนกัน ต่างที่ปริมาณ token)

กรอกบัตรเพื่อชำระเงิน

กรอกข้อมูล บัตรเดบิต/เครดิต เพื่อชำระค่าแพ็กเกจ → กด Update / Subscribe

ตรวจว่าขึ้น Pro plan แล้ว

คลิกอวตาร์ → Settings → Billing → ตรวจว่าขึ้นว่า Pro plan (หรือ Max) และมีข้อมูลใบเสร็จ — เป็นอันเรียบร้อย ✓

🔑บัญชีนี้คือกุญแจของ Claude Code เวลาล็อกอินบนคอม — จำอีเมล+รหัสผ่านให้ดี เก็บที่ปลอดภัย
03
ติดตั้งลงคอม · Windows

ลงโปรแกรมที่ต้องใช้ 3 ตัว

ทำครั้งเดียวจบ · ทุกอย่างพิมพ์ใน PowerShell (กดปุ่ม Windows → พิมพ์ “PowerShell” → เปิด)

⌨️
PowerShell คืออะไร? คือหน้าต่างดำ ๆ ของ Windows ที่เราพิมพ์คำสั่งให้คอมทำงาน (เรียกว่า “เทอร์มินอล”) · เปิดโดยกดปุ่ม Windows ที่คีย์บอร์ด → พิมพ์ PowerShell → กดเปิด

ติดตั้ง Node.js — ตัวช่วยเบื้องหลัง

เข้า nodejs.org → โหลดปุ่มเขียว “LTS” → เปิดไฟล์ที่โหลด → กด Next ไปเรื่อย ๆ จนเสร็จ (จำเป็นสำหรับติดตั้งอีก 2 ตัวถัดไป)

💡Node.js = โปรแกรมพื้นฐานที่ทำให้ติดตั้งเครื่องมือตัวอื่น (Claude Code, clasp) ได้ · LTS = เวอร์ชันเสถียร แนะนำให้โหลดอันนี้
📸ใส่ภาพ: หน้า nodejs.org ปุ่ม LTS สีเขียว

เช็คว่าลงสำเร็จ — เปิด PowerShell แล้วพิมพ์:

> node -v

ถ้าขึ้นเลขเวอร์ชัน (เช่น v22.x.x) = ผ่าน ✓

ติดตั้ง Claude Code

ใน PowerShell พิมพ์คำสั่งนี้แล้วกด Enter (รอสักครู่ให้ติดตั้งเสร็จ):

> npm install -g @anthropic-ai/claude-code
💡npm = ตัวติดตั้งโปรแกรม (มากับ Node.js) · install -g = ติดตั้งแบบใช้ได้ทั้งเครื่อง · ตัวยาว ๆ ท้ายสุดคือชื่อโปรแกรม Claude Code

ติดตั้ง clasp — สะพานส่งโค้ดขึ้น Google

ตัวนี้คือที่ Claude ใช้ “อัปโค้ดขึ้นระบบ” ให้ พิมพ์:

> npm install -g @google/clasp
🌉clasp = เครื่องมือของ Google ที่ส่งโค้ดจากคอมเราขึ้นไปที่ Apps Script (ที่บอทรันอยู่) — เหมือน “สะพาน” เชื่อมคอมกับระบบจริง

ล็อกอิน Claude Code ครั้งแรก

พิมพ์ claude เพื่อเปิดโปรแกรม → พิมพ์ /login → เบราว์เซอร์จะเด้งขึ้นมา → ล็อกอินด้วยบัญชี Claude Pro ที่สมัครไว้ → เสร็จแล้วกลับมาที่ PowerShell

> claude
🔑ทำครั้งเดียวพอ — ครั้งต่อ ๆ ไปพิมพ์ claude เปิดใช้ได้เลย ไม่ต้องล็อกอินซ้ำ
📸ใส่ภาพ: หน้าจอ Claude Code หลังเปิดสำเร็จให้เห็นข้อความต้อนรับ
🪟
เคล็ดลับ Windows: ถ้าพิมพ์คำสั่งแล้วขึ้น “ไม่รู้จักคำสั่ง” ให้ ปิด PowerShell แล้วเปิดใหม่ หนึ่งครั้งหลังลง Node.js เสร็จ — โปรแกรมจะเห็นคำสั่งใหม่

รับมอบโปรเจกต์

รับไฟล์ระบบ + กุญแจจาก Moodata

สิ่งที่ Moodata ส่งมอบให้ พร้อมนำโปรเจกต์ลงเครื่องเพื่อให้ Claude Code เปิดทำงานได้

📦

สิ่งที่ได้รับ (ระบบ)

โฟลเดอร์โปรเจกต์ kanchuda-order — โค้ด 13 ไฟล์ + คู่มือระบบ (SKILL.md) ที่ทำให้ Claude เข้าใจกานต์ชุดาทันที · ส่งผ่านลิงก์ GitHub หรือไฟล์ zip

🔑

สิ่งที่ได้รับ (กุญแจ)

รหัสประจำโปรเจกต์ Apps Script (scriptId) · รหัส deployment สำหรับอัปเดตบอท · สิทธิ์บัญชี Google ให้เข้าถึงระบบ · ตำแหน่ง Token ลับต่าง ๆ

วางโฟลเดอร์ไว้ที่หาง่าย

แตกไฟล์ / วางโฟลเดอร์ kanchuda-order ไว้ที่ไดรฟ์ C หรือ Desktop เช่น C:\kanchuda-order

เปิด PowerShell “ในโฟลเดอร์นั้น”

เปิดโฟลเดอร์ใน File Explorer → คลิกช่องที่อยู่ด้านบน → พิมพ์ powershell แล้ว Enter (PowerShell จะเปิดตรงโฟลเดอร์นี้พอดี) หรือใช้คำสั่ง:

> cd C:\kanchuda-order
💡cd = คำสั่ง “ย้ายเข้าไปในโฟลเดอร์” · Claude Code ต้องเปิด ในโฟลเดอร์โปรเจกต์ ถึงจะเห็นไฟล์ระบบทั้งหมด
🗝️
โมเดลส่งมอบ: Google Sheet + โฟลเดอร์รายงาน เป็น “ของกานต์ชุดา” · ตัวโค้ด/บอทที่ Moodata เคยดูแล จะโอนสิทธิ์มาให้บัญชี Google ของคุณเข้าถึงและอัปเดตได้เอง — Moodata นัดทำขั้นตอนนี้ให้ในวันส่งมอบ
05
เชื่อมต่อ & ทดสอบ

ต่อ Claude Code เข้ากับระบบจริง

ล็อกอิน Google ให้ clasp แล้วลองสั่ง Claude อ่านระบบ 1 ครั้ง เพื่อยืนยันว่าทุกอย่างพร้อม

ล็อกอิน Google ให้ clasp

พิมพ์คำสั่งนี้ → เบราว์เซอร์เด้ง → เลือก บัญชี Google ที่มีสิทธิ์ในระบบ (บัญชีที่ Moodata โอนสิทธิ์ให้) → Allow

> clasp login

เปิด Claude Code ในโปรเจกต์

อยู่ในโฟลเดอร์ kanchuda-order แล้วพิมพ์:

> claude

ลองสั่งอ่านระบบ 1 ครั้ง (เทสว่าพร้อม)

พิมพ์เป็นภาษาไทยได้เลย:

คุณ › ช่วยสรุปว่าระบบกานต์ชุดาทำอะไรได้บ้าง

ถ้า Claude ตอบได้ถูกต้องว่าเป็นระบบเก็บออเดอร์ผ่าน LINE + ออกรายงาน WMS/PEAK = เชื่อมต่อสำเร็จ ✓

อัปโค้ดขึ้นระบบ — สั่ง Claude ทำให้ได้เลย

เมื่อพอใจกับที่แก้แล้ว พิมพ์สั่งต่อได้เลยว่า “อัปโค้ดขึ้นระบบให้หน่อย” — Claude จะรันคำสั่งส่งขึ้น Google ให้ (เบื้องหลังคือ clasp push)

⚠️
ถ้าเป็นการแก้ที่เกี่ยวกับ การรับข้อความจากไลน์ (เช่น การยกเลิก/ลงทะเบียน) ต้อง “redeploy” โดยคง URL เดิม — บอก Claude ว่า “ช่วย redeploy โดยใช้ deployment เดิม” พร้อมส่งรหัส deployment ที่ Moodata ให้ · อย่าสร้าง deployment ใหม่ เพราะ URL จะเปลี่ยนแล้วบอทจะเงียบ

เก็บงานให้เรียบร้อย (ถ้าใช้ GitHub)

สั่ง Claude ว่า “ช่วย commit แล้ว push ขึ้น GitHub ให้” เพื่อสำรองโค้ดเวอร์ชันล่าสุดไว้เสมอ

🧭
ไม่แน่ใจตรงไหน ถาม Claude ได้เลย — พิมพ์ว่า “ขั้นตอนต่อไปต้องทำอะไร” หรือ “อธิบายให้ละเอียดกว่านี้” ได้ทุกเมื่อ Claude เป็นผู้ช่วยที่คุยโต้ตอบได้

ให้ใช้ “redeploy เดิม” เสมอ (บอก Claude ให้ใช้ deployment ID ที่ได้รับ) · ถ้าสร้างใหม่ URL จะเปลี่ยน → บอทเงียบทั้งระบบ

🔗

Webhook URL ต้องมี ?token= ต่อท้ายเสมอ

ถ้าหาย บอทจะขึ้น “อ่านแล้ว” ในกลุ่มแต่ไม่บันทึกออเดอร์ — เป็นจุดแรกที่ต้องเช็คเวลาบอทเงียบ

🔒

เก็บ Token เป็นความลับ

LINE_TOKEN และ WEBHOOK_TOKEN คือกุญแจบอท — อย่าส่งในแชทสาธารณะ หรือ push ขึ้น GitHub แบบเปิด

คำถามที่พบบ่อย

FAQ & แก้ปัญหา

คำถามที่มักเจอตอนรับมอบระบบ — คลิกที่คำถามเพื่อดูคำตอบ

ต้องเขียนโปรแกรมเป็นไหม?
ไม่ต้องค่ะ — พิมพ์สั่ง Claude เป็นภาษาไทยได้เลย เช่น “ช่วยแก้ข้อความแจ้งตัดรอบ” Claude จะจัดการโค้ดให้เอง เราแค่ตรวจและกดยืนยัน
ถ้าสั่งผิด ระบบจะพังทันทีไหม?
ไม่ทันทีค่ะ — Claude แก้ไฟล์ที่อยู่ ในคอมเราก่อน ต้องสั่ง “อัปขึ้นระบบ” (clasp push) ถึงจะมีผลจริง · และถ้าใช้ GitHub มีสำรองย้อนเวอร์ชันเก่าได้เสมอ — ช่วง 1–2 ครั้งแรกให้ Moodata คอยดูด้วยจะอุ่นใจ
Claude Code ต่างจาก claude.ai (เว็บแชท) ยังไง?
claude.ai = คุยแชทอย่างเดียว (ช่วยคิด/ร่าง) · Claude Code = ทำงานบนคอมได้จริง — อ่านไฟล์โค้ด แก้ไฟล์ และรันคำสั่งอัปขึ้นระบบให้ · งานดูแลบอทใช้ Claude Code
ต้องเปิดคอมทิ้งไว้ตลอดให้บอททำงานไหม?
ไม่ต้องค่ะ — บอทรันอยู่บน Google ตลอด 24 ชม. เอง · เปิดคอม + Claude Code เฉพาะตอนที่ อยากแก้/อัปเดตระบบ เท่านั้น
แก้เสร็จแล้วบอทเงียบ ไม่ตอบในกลุ่ม ทำยังไง?
มัก 2 สาเหตุ: (1) แก้ระบบไลน์แล้วยังไม่ได้ redeploy · (2) สร้าง deployment ใหม่จน URL เปลี่ยน → บอก Claude ว่า “ช่วย redeploy โดยใช้ deployment เดิม” · ถ้ายังไม่หาย เช็ค ?token= ใน webhook หรือติดต่อ Moodata
มีค่าใช้จ่ายอะไรบ้าง?
หลัก ๆ คือ Claude Pro $20/เดือน · ส่วน Google Apps Script + LINE Messaging API ใช้ฟรีในโควต้าปกติของร้าน
ถ้าไม่อยากดูแลเองแล้ว ให้ Moodata ทำแทนได้ไหม?
ได้ค่ะ — เลือก “แบบ B” (Moodata ดูแลให้ ฿2,500/เดือน) ที่หัวข้อ “2 ทางเลือกในการดูแลต่อ” ด้านบน · เปลี่ยนใจไปมาได้ตลอด

คำศัพท์เทคนิคที่ควรรู้

ศัพท์ที่เจอในคู่มือ อธิบายง่าย ๆ

ไม่ต้องจำหมด — เปิดดูตอนเจอคำที่ไม่คุ้นก็พอ

PowerShell
หน้าต่างพิมพ์คำสั่งของ Windows (เรียกว่า “เทอร์มินอล”) ที่เราพิมพ์สั่งให้คอมทำงาน
Node.js
โปรแกรมพื้นฐานที่ทำให้ติดตั้งเครื่องมือตัวอื่น (Claude Code, clasp) ได้
npm
ตัวติดตั้งโปรแกรม มากับ Node.js — ใช้พิมพ์ npm install เพื่อลงเครื่องมือ
Claude Code
Claude เวอร์ชันที่ทำงานบนคอม อ่าน/แก้ไฟล์โค้ด และรันคำสั่งให้ได้
clasp
เครื่องมือของ Google ที่ส่งโค้ดจากคอมขึ้นไปที่ Apps Script (สะพานเชื่อมคอม↔ระบบจริง)
Apps Script
ที่ที่โค้ดบอทกานต์ชุดารันอยู่ (บริการของ Google ผูกกับ Google Sheet)
deploy / deployment
การ “เผยแพร่” โค้ดเวอร์ชันใหม่ให้ใช้งานจริง · redeploy = อัปเดตโดยคง URL เดิม (สำคัญกับบอท)
webhook
ช่องทางที่ LINE ส่งข้อความในกลุ่มมาให้บอท — ต้องมี ?token= ต่อท้าย URL เสมอ
scriptId
รหัสประจำโปรเจกต์ Apps Script — clasp ใช้รหัสนี้รู้ว่าจะส่งโค้ดไปที่ไหน
Token
รหัสลับ (กุญแจ) ของบอท เช่น LINE_TOKEN · เก็บเป็นความลับ ห้ามเผยแพร่
repo / GitHub
ที่เก็บโค้ดออนไลน์ ไว้สำรองและดูประวัติการแก้ — สั่ง Claude “commit + push” เพื่อบันทึก
clasp push
คำสั่ง “อัปโค้ดขึ้นระบบ” — สั่ง Claude เป็นภาษาไทยว่า “อัปขึ้นระบบให้หน่อย” ก็ได้
