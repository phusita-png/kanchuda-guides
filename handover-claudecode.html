Claude Code คืออะไร แล้วช่วยดูแลบอทได้ยังไง

ผู้ช่วย AI ที่ทำงานบนคอมของคุณ — อ่านโค้ดระบบกานต์ชุดาทั้งหมดได้ แก้ให้ได้ แล้วอัปขึ้นระบบให้ด้วย โดยคุณแค่พิมพ์สั่งเป็นภาษาไทย

02
สมัคร & เลือกแพ็กเกจ

สมัครบัญชี Claude และเปิดแพ็กเกจ Pro

ใช้อีเมลอะไรก็ได้ · แพ็กเกจ Pro $20/เดือน ใช้ Claude Code ได้เพียงพอสำหรับดูแลบอทกานต์ชุดา

เปิดเว็บ claude.ai แล้วกด Sign up

เข้า claude.ai → สมัครด้วยอีเมลบริษัท หรือปุ่ม “Continue with Google” (เร็วสุด) → ยืนยันอีเมล

อัปเกรดเป็น Claude Pro

กดชื่อบัญชีมุมล่างซ้าย → Upgrade / Plans → เลือก Pro — $20/เดือน → กรอกบัตร → ยืนยัน

✨
ทำไม Pro ก็พอ: การดูแลบอทเป็นการแก้เป็นครั้งคราว ไม่ได้เขียนใหม่ทั้งวัน โควต้าของ Pro รับไหวสบาย — ถ้าอนาคตแก้บ่อยมากจนโควต้าหมดเร็ว ค่อยขยับเป็น Max ทีหลังได้

จำอีเมล + รหัสผ่านให้ดี

บัญชีนี้คือกุญแจของ Claude Code เวลาล็อกอินบนคอม — เก็บไว้ที่ปลอดภัย ใช้ได้ทั้งบนเว็บและในคอม

03
ติดตั้งลงคอม · Windows

ลงโปรแกรมที่ต้องใช้ 3 ตัว

ทำครั้งเดียวจบ · ทุกอย่างพิมพ์ใน PowerShell (กดปุ่ม Windows → พิมพ์ “PowerShell” → เปิด)

ติดตั้ง Node.js — ตัวช่วยเบื้องหลัง

เข้า nodejs.org → โหลดปุ่มเขียว “LTS” → เปิดไฟล์ที่โหลด → กด Next ไปเรื่อย ๆ จนเสร็จ (จำเป็นสำหรับติดตั้งอีก 2 ตัวถัดไป)

เช็คว่าลงสำเร็จ — เปิด PowerShell แล้วพิมพ์:

> node -v

ถ้าขึ้นเลขเวอร์ชัน (เช่น v22.x.x) = ผ่าน ✓

ติดตั้ง Claude Code

ใน PowerShell พิมพ์คำสั่งนี้แล้วกด Enter (รอสักครู่):

> npm install -g @anthropic-ai/claude-code

ติดตั้ง clasp — สะพานส่งโค้ดขึ้น Google

ตัวนี้คือที่ Claude ใช้ “อัปโค้ดขึ้นระบบ” ให้ พิมพ์:

> npm install -g @google/clasp

ล็อกอิน Claude Code ครั้งแรก

พิมพ์ claude เพื่อเปิดโปรแกรม → พิมพ์ /login → เบราว์เซอร์จะเด้งขึ้นมา → ล็อกอินด้วยบัญชี Claude Pro ที่สมัครไว้ → เสร็จแล้วกลับมาที่ PowerShell

> claude
🪟
เคล็ดลับ Windows: ถ้าพิมพ์คำสั่งแล้วขึ้น “ไม่รู้จักคำสั่ง” ให้ ปิด PowerShell แล้วเปิดใหม่ หนึ่งครั้งหลังลง Node.js เสร็จ — โปรแกรมจะเห็นคำสั่งใหม่
04
รับมอบโปรเจกต์

รับไฟล์ระบบ + กุญแจจาก Moodata

สิ่งที่ Moodata ส่งมอบให้ พร้อมนำโปรเจกต์ลงเครื่องเพื่อให้ Claude Code เปิดทำงานได้

📦

สิ่งที่ได้รับ (ระบบ)

โฟลเดอร์โปรเจกต์ kanchuda-order — โค้ด 13 ไฟล์ + คู่มือระบบ (SKILL.md) ที่ทำให้ Claude เข้าใจกานต์ชุดาทันที · ส่งผ่านลิงก์ GitHub หรือไฟล์ zip

🔑

สิ่งที่ได้รับ (กุญแจ)

รหัสประจำโปรเจกต์ Apps Script (scriptId) · รหัส deployment สำหรับอัปเดตบอท · สิทธิ์บัญชี Google ให้เข้าถึงระบบ · ตำแหน่ง Token ลับต่าง ๆ

วางโฟลเดอร์ไว้ที่หาง่าย

แตกไฟล์ / วางโฟลเดอร์ kanchuda-order ไว้ที่ไดรฟ์ C หรือ Desktop เช่น C:\kanchuda-order

เปิด PowerShell “ในโฟลเดอร์นั้น”

เปิดโฟลเดอร์ใน File Explorer → คลิกช่องที่อยู่ด้านบน → พิมพ์ powershell แล้ว Enter (PowerShell จะเปิดตรงโฟลเดอร์นี้พอดี) หรือใช้คำสั่ง:

> cd C:\kanchuda-order
🗝️
โมเดลส่งมอบ: Google Sheet + โฟลเดอร์รายงาน เป็น “ของกานต์ชุดา” · ตัวโค้ด/บอทที่ Moodata เคยดูแล จะโอนสิทธิ์มาให้บัญชี Google ของคุณเข้าถึงและอัปเดตได้เอง — Moodata นัดทำขั้นตอนนี้ให้ในวันส่งมอบ
05
เชื่อมต่อ & ทดสอบ

ต่อ Claude Code เข้ากับระบบจริง

ล็อกอิน Google ให้ clasp แล้วลองสั่ง Claude อ่านระบบ 1 ครั้ง เพื่อยืนยันว่าทุกอย่างพร้อม

ล็อกอิน Google ให้ clasp

พิมพ์คำสั่งนี้ → เบราว์เซอร์เด้ง → เลือก บัญชี Google ที่มีสิทธิ์ในระบบ (บัญชีที่ Moodata โอนสิทธิ์ให้) → Allow

> clasp login

เปิด Claude Code ในโปรเจกต์

อยู่ในโฟลเดอร์ kanchuda-order แล้วพิมพ์:

> claude

ลองสั่งอ่านระบบ 1 ครั้ง (เทสว่าพร้อม)

พิมพ์เป็นภาษาไทยได้เลย:

คุณ › ช่วยสรุปว่าระบบกานต์ชุดาทำอะไรได้บ้าง

ถ้า Claude ตอบได้ถูกต้องว่าเป็นระบบเก็บออเดอร์ผ่าน LINE + ออกรายงาน WMS/PEAK = เชื่อมต่อสำเร็จ ✓

06
สอนใช้งานจริง

วิธีสั่งงาน Claude Code ในชีวิตจริง

จังหวะเดียวกันเสมอ: บอกสิ่งที่อยากได้ → Claude แก้ให้ดู → สั่งอัปขึ้นระบบ — ลองจากตัวอย่างจริงของกานต์ชุดา

🔍

ให้ช่วยหาสาเหตุ

“ทำไมรายงานไม่ขึ้นราคาบางออเดอร์ ช่วยหาสาเหตุให้”

Claude ไล่โค้ดหาจุดที่ผิดแล้วอธิบายให้ฟังก่อนแก้

➕

ให้เพิ่มความสามารถ

“ให้บอทรู้จักคำว่า จ่ายแล้ว ว่าเป็นการโอนเงินด้วย”

Claude แก้ไฟล์ให้ แล้วสรุปว่าแก้อะไรไปบ้าง

📖

ให้ช่วยอธิบาย

“ไฟล์ 10_Notify ทำหน้าที่อะไร อธิบายแบบเข้าใจง่าย”

เหมาะเวลาอยากเข้าใจระบบก่อนแตะ

อัปโค้ดขึ้นระบบ — สั่ง Claude ทำให้ได้เลย

เมื่อพอใจกับที่แก้แล้ว พิมพ์สั่งต่อได้เลยว่า “อัปโค้ดขึ้นระบบให้หน่อย” — Claude จะรันคำสั่งส่งขึ้น Google ให้ (เบื้องหลังคือ clasp push)

⚠️
ถ้าเป็นการแก้ที่เกี่ยวกับ การรับข้อความจากไลน์ (เช่น การยกเลิก/ลงทะเบียน) ต้อง “redeploy” โดยคง URL เดิม — บอก Claude ว่า “ช่วย redeploy โดยใช้ deployment เดิม” พร้อมส่งรหัส deployment ที่ Moodata ให้ · อย่าสร้าง deployment ใหม่ เพราะ URL จะเปลี่ยนแล้วบอทจะเงียบ

เก็บงานให้เรียบร้อย (ถ้าใช้ GitHub)

สั่ง Claude ว่า “ช่วย commit แล้ว push ขึ้น GitHub ให้” เพื่อสำรองโค้ดเวอร์ชันล่าสุดไว้เสมอ

🧭
ไม่แน่ใจตรงไหน ถาม Claude ได้เลย — พิมพ์ว่า “ขั้นตอนต่อไปต้องทำอะไร” หรือ “อธิบายให้ละเอียดกว่านี้” ได้ทุกเมื่อ Claude เป็นผู้ช่วยที่คุยโต้ตอบได้
07
เช็คลิสต์ส่งมอบ

ตรวจให้ครบก่อนบอกว่า “รับมอบเรียบร้อย”

ติ๊กทีละข้อ — ครบเมื่อไหร่ ระบบกานต์ชุดาก็อยู่ในมือคุณเต็มตัว

ความคืบหน้า 0 / 8

⚠️ 3 ข้อห้ามพลาด (จำให้ขึ้นใจ)

🚫

อย่าสร้าง deployment ใหม่ตอนแก้ระบบไลน์

ให้ใช้ “redeploy เดิม” เสมอ (บอก Claude ให้ใช้ deployment ID ที่ได้รับ) · ถ้าสร้างใหม่ URL จะเปลี่ยน → บอทเงียบทั้งระบบ

🔗

Webhook URL ต้องมี ?token= ต่อท้ายเสมอ

ถ้าหาย บอทจะขึ้น “อ่านแล้ว” ในกลุ่มแต่ไม่บันทึกออเดอร์ — เป็นจุดแรกที่ต้องเช็คเวลาบอทเงียบ

🔒

เก็บ Token เป็นความลับ

LINE_TOKEN และ WEBHOOK_TOKEN คือกุญแจบอท — อย่าส่งในแชทสาธารณะ หรือ push ขึ้น GitHub แบบเปิด
